## Executive Summary

**Problem:** PPO cannot improve beyond the BC baseline. ExplVar stays negative
in recent runs (046/047/048), and even when positive (run 043), win rate doesn't
improve. BC knowledge erodes steadily in ALL runs.

**CRITICAL NEW FINDING:** The critic warmstart at `models/bc_warmstart.zip`
ALREADY WORKS — ExplVar = 0.99 on its training data. PPO then DESTROYS it
within 23K steps (run 047: ExplVar 0.99 → -0.26). Weight diffs are tiny
(mean abs 0.02) yet catastrophic.

**Root cause (verified with evidence):**
The value function is properly initialized by `warmstart_critic.py`. The problem
is that PPO's online value updates — 10 epochs per rollout at LR 3e-4 on only
8192 samples — overfit and destroy the generalization in ~3-4 rollouts. Once the
value function is ruined, advantage estimates become noise, eroding BC knowledge.

**Why run 043 was different:** LR=1e-4 constant (3x lower), [64,64] network
(4x fewer params = harder to overfit), batch_size=64. All three factors reduce
the rate at which PPO destroys the value function.

**Fix (implemented):**
1. **Actor freeze** for 50K steps — prevents state distribution shift while
   value function adapts to online GAE targets
2. **n_epochs=3** (was 10) — prevents per-rollout overfitting that destroys
   the warmstarted value function
3. **LR=1e-4 constant** (was 3e-4→1e-4) — smaller updates preserve warmstart
4. **clip_range=0.1** (was 0.2) — conservative policy updates post-unfreeze
5. **Weight copy** in BC script — copies policy_net→value_net as additional
   safety net for value initialization

**Validation experiments running** (results pending, plan proceeds regardless
based on warmstart investigation evidence).

**File:** `scripts/behavioral_cloning.py`

weights to the value_net:

```python
# Copy BC-trained actor features to initialize the value function.
# policy_net and value_net have identical architecture [256, 128],

with torch.no_grad():
```

### Step 4: Add actor freeze/unfreeze to train.py

**File:** `src/train.py`

Add the freeze/unfreeze utility functions from the previous plan (these are
still correct):

```python
def _freeze_actor(model) -> None:
    """Freeze all actor parameters — only the critic trains."""
    frozen = 0
    for param in model.policy.mlp_extractor.policy_net.parameters():
        param.requires_grad = False
        frozen += param.numel()
    for param in model.policy.action_net.parameters():
        param.requires_grad = False
        frozen += param.numel()
    print(f"  [Actor Freeze] Froze {frozen:,} actor parameters")

def _unfreeze_actor(model) -> None:
    """Unfreeze all actor parameters — full PPO training resumes."""
    unfrozen = 0
    for param in model.policy.mlp_extractor.policy_net.parameters():
        param.requires_grad = True
        unfrozen += param.numel()
    for param in model.policy.action_net.parameters():
        param.requires_grad = True
        unfrozen += param.numel()
    print(f"  [Actor Unfreeze] Unfroze {unfrozen:,} actor parameters")
```

Add a critic warmup phase before the curriculum loop (code from previous plan,
with one key change: set `gae_lambda=1.0` during warmup):

```python
CRITIC_WARMUP_STEPS = 50_000   # Reduced from 100K — weight copy means faster convergence

```

During warmup, temporarily override gae_lambda:
```python
# During warmup, use MC returns (gae_lambda=1.0) so the value function

model.gae_lambda = 1.0
```

## 3.5 Escalation Plans

### Escalation A: ExplVar stays negative during warmup (despite weight copy)

**Diagnosis:** The copied features aren't useful for value prediction, OR the
value head can't learn on top of them.

**Fix:** Try `gae_lambda=1.0` for the ENTIRE warmup (already the default in
the plan — this escalation means extending the warmup):

```python
CRITIC_WARMUP_STEPS = 100_000  # Double the warmup
```

If still negative after 100K: the value function fundamentally can't learn from
this obs space. Try:
- Wider value network: `vf=[512, 256, 128]` (but requires re-running BC with new
  policy_kwargs to get the right shapes for weight copy — the vf would now be
  different from pi, so only copy the first layer [1559→512] from policy's
  [1559→256])
- Terminal-only reward: `fainted_value=0.0, victory_value=1.0` (simpler target
  for value function — just predict win probability)

### Escalation B: ExplVar drops negative after unfreezing

**Diagnosis:** Policy changes too fast, destabilizing the critic.

**Fix:** Gradual unfreeze:
```python
# Phase 1: Unfreeze only the action head (Linear(128, 10))
for param in model.policy.action_net.parameters():
    param.requires_grad = True
model.learn(total_timesteps=50_000, ...)

# Phase 2: Unfreeze the full actor MLP
for param in model.policy.mlp_extractor.policy_net.parameters():
    param.requires_grad = True
```

Also reduce clip_range to 0.05 and LR to 5e-5 for the first 50K post-unfreeze.

### Escalation C: ExplVar positive but win rate doesn't improve

**Diagnosis:** Same as run 043 — value function works but PPO can't improve.
This is the hardest failure mode.

**Fix options (try in order):**
1. **Longer rollouts:** `n_steps=4096` (more data per PPO update = lower gradient variance)
2. **More envs:** `N_ENVS=8` (more diverse data per rollout)
3. **Denser reward:** Add `hp_value=0.1` back (more frequent reward signal)
4. **Separate actor/critic LR:**
   ```python
   actor_params = list(model.policy.mlp_extractor.policy_net.parameters()) + \
                  list(model.policy.action_net.parameters())
   critic_params = list(model.policy.mlp_extractor.value_net.parameters()) + \
                   list(model.policy.value_net.parameters())
   model.policy.optimizer = torch.optim.Adam([
       {"params": actor_params, "lr": 5e-5},
       {"params": critic_params, "lr": 3e-4},
   ])
   ```
   This lets the critic adapt 6x faster than the actor changes.

### Escalation D: Win rate declines over training (BC erosion continues)

**Diagnosis:** PPO updates are net-negative. The policy gradient signal is too
noisy to provide useful direction.

**Fix options:**
1. **BC regularization:** Add KL penalty against the BC policy distribution.
   Before unfreezing, save BC action logits on a validation set. During
   training, add `kl_coef * KL(π_current || π_BC)` to the loss. Start with
   kl_coef=0.1, decay to 0 over 200K steps.
2. **AWAC-style training:** Instead of PPO, use Advantage-Weighted Actor-Critic
   which is specifically designed for learning from demonstrations. It
   naturally prevents policy degradation by weighting updates by advantage.
3. **Reduce to simpler problem:** Train against MaxDamage only (single env,
   not mixed league). If PPO can improve against a single opponent, the
   mixed league problem is opponent diversity, not the algorithm.
